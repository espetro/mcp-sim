# v0.1.1 — integration-test hardening

**Released:** 2026-07-05 (planned)
**Tag:** `v0.1.1` on `main` (planned)

Patch release. Fixes the bugs surfaced by the v0.1.0 integration test.

## Fixed

- **Android emulator killed by request ctx** — `boot_device` returned success but the emulator died ~10s later when adb showed `offline`. Cause: `exec.CommandContext(reqCtx)` propagated SIGKILL when the SDK cancelled the request. Emulator now spawned with `context.Background()` so it survives the request lifecycle.
- **Stdio inherited from server** — emulator/agent-device stdio redirected to `io.Discard` to avoid SIGPIPE on backgrounded servers.
- **`agent-device proxy` subcommand doesn't exist** — refactored `controllers/agentdevice` as a presence checker. The original "proxy launcher" concept was based on a wrong assumption about agent-device's CLI; mcp-sim now just reports whether the binary is available, leaving the actual MCP server invocation to the agent client.
- **`open_url` errors had no context** — both iOS and Android now use `CombinedOutput()` and include stderr so callers see the actual reason an intent failed (e.g. "Activity not started, unable to resolve Intent" on AVDs without a browser).
- **`stop_device` didn't reliably terminate Android emulator** — now tracks spawned PIDs and SIGTERMs the process group directly via `syscall.Kill(-pid, SIGTERM)`. `adb emu kill` becomes a fallback.
- **`await_ready` default timeout too short** — bumped from 60s to 180s. iOS Simulator boot on slower Macs takes ~90s.
- **`boot_device` default timeout too short** — bumped from 60s to 120s in `lifecycle.BootDevice`.
- **AGENTS.md had stale release content** — removed the `## Current release` section. Release docs now live in `docs/releases/` and AGENTS.md references them via progressive disclosure.

## How to upgrade

```bash
brew upgrade mcp-sim
# or
go install github.com/espetro/mcp-sim/cmd/mcp-sim@v0.1.1
```

No config migration required.

## Known caveats (still)

- **Android AVDs without a browser app** still reject `open_url https://...`. We surface this as a clear "Activity not started" error instead of a generic exit code, but the user must install Chrome or another browser on the AVD. Tracked for v0.1.2.
- **`agent-device` controller is presence-only in v0.1.x.** Future versions can spawn `agent-device mcp` for HTTP-attached MCP.
